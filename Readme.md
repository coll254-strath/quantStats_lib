# Mag 7 Quant Analysis — README

**A QuantStats-powered performance and risk analytics pipeline for the "Magnificent 7" equities, benchmarked against the S&P 500.**

---

## 1. What This Code Does (One Sentence)

It takes daily price history for AAPL, MSFT, GOOGL, AMZN, META, NVDA, and TSLA, and uses **QuantStats** to turn raw returns into institutional-grade performance metrics, risk diagnostics, and executive-ready visual reports.

---

## 2. The Framework: 3 Layers

| Layer | Role | Owned By |
|---|---|---|
| **Data Layer** | Pulls daily adjusted close prices | `yfinance` |
| **Analytics Layer** | Converts prices → returns → performance/risk metrics | **`QuantStats`** |
| **Reporting Layer** | Renders metrics into dashboards + tearsheets | **`QuantStats`** + `matplotlib` |

QuantStats is the engine of this pipeline — everything from Layer 2 onward runs through it. The sections below focus entirely on what QuantStats is doing and why each metric matters.

---

## 3. What QuantStats Calculates — And Why It Matters to the Business

| QuantStats Function | Metric | Executive Translation |
|---|---|---|
| `qs.stats.cagr()` | Compound Annual Growth Rate | "How much did this stock actually grow, per year, on average?" |
| `qs.stats.volatility()` | Annualized Volatility | "How bumpy was the ride to get that growth?" |
| `qs.stats.sharpe()` | Sharpe Ratio | "Was the return worth the risk taken?" — higher is better |
| `qs.stats.sortino()` | Sortino Ratio | Same as Sharpe, but only penalizes *downside* risk — a fairer lens for volatile growth stocks like NVDA/TSLA |
| `qs.stats.max_drawdown()` | Max Drawdown | "What's the worst-case peak-to-trough loss an investor would have sat through?" |
| `qs.stats.greeks()` | Beta vs S&P 500 | "Does this stock amplify or dampen market moves?" |
| `qs.stats.to_drawdown_series()` | Drawdown Timeline | Visual — shows *when* the pain happened, not just how deep |

Every number in the executive summary traces back to one of these seven functions. Nothing is hand-calculated — QuantStats' internal math is used end-to-end, which means the metrics are auditable against a well-established open-source library rather than custom formulas.

---

## 4. What QuantStats Generates — The Two Deliverables

### A. The Tearsheet (`qs.reports.html()`)
One HTML file per stock, benchmarked against the S&P 500. This is the **deep-dive artifact** — hand it to an analyst, not a room full of executives. It includes cumulative returns, rolling volatility, monthly return heatmaps, drawdown periods, and a full metrics table, all auto-generated by QuantStats from a single function call.

### B. The Metrics Table (`compute_metrics_table()`)
A condensed, one-row-per-stock summary built by calling the QuantStats functions above in a loop and assembling the results into a single table, sorted by Sharpe Ratio. This is the **boardroom artifact** — one glance tells you which of the seven names delivered the best risk-adjusted return over the period.

## 7. Known Issue: Arial Font Warning

QuantStats' default plot theme references the `Arial` font, which isn't installed on Linux-based environments (including Colab) by default. This is **cosmetic only** — matplotlib falls back to DejaVu Sans and all charts render correctly. Fix by adding, before any plotting:

```python
plt.rcParams['font.family'] = 'DejaVu Sans'
```
