### Detailed Explanation of the Pipeline

This pipeline is designed for a systematic quantitative analysis of a set of stock tickers, comparing their performance against a benchmark. It follows a 5-stage workflow:

#### Stage 0: Configuration

This section sets up global variables and environment settings:

*   **`TICKERS`**: A list of stock symbols (e.g., Apple, Microsoft, Google) to be analyzed.
*   **`BENCHMARK`**: The benchmark index (e.g., S&P 500, represented by 'SPY') for comparison.
*   **`LOOKBACK_YEARS`**: The historical period (in years) for which to fetch data.
*   **`OUTPUT_DIR`**: The directory where all generated reports and plots will be saved.
*   **`TEARSHEET_DIR`**: A sub-directory within `OUTPUT_DIR` specifically for individual tearsheet reports.
*   **`os.makedirs(TEARSHEET_DIR, exist_ok=True)`**: Creates the output directories if they don't already exist.
*   **`sns.set_theme(...)`**: Configures the aesthetic style for `seaborn` plots.
*   **`COLOR_PALETTE`**: Defines a color scheme for visualizations.

#### Stage 1: Acquire Data

This stage focuses on gathering the necessary financial data.

*   **`fetch_price_data(tickers, benchmark, years)`**: This function uses the `yfinance` library to download historical adjusted close prices for all specified tickers and the benchmark. It returns a Pandas DataFrame where each column represents a stock's adjusted close prices over the `LOOKBACK_YEARS` period.
*   **`fetch_fundamentals(tickers)`**: This function fetches fundamental data (like P/E ratio, Market Cap, EPS, ROE, Debt/Equity, Beta, Dividend) for each ticker using the `finvizfinance` library. It iterates through each ticker, retrieves its fundamental snapshot, and compiles it into a Pandas DataFrame. Error handling is included to manage cases where data might not be available for a specific ticker.

#### Stage 2: Structure Data

Once acquired, the raw price data needs to be structured for analysis.

*   **`build_returns_matrix(price_df)`**: This function takes the DataFrame of historical prices and converts them into daily simple returns using `pct_change()`. It then removes any initial `NaN` values that result from this calculation.

#### Stage 3 & 4: Measure & Benchmark Performance

These stages calculate key performance metrics and compare them against the chosen benchmark.

*   **`compute_metrics_table(returns, tickers, benchmark)`**: This function calculates a comprehensive set of risk and return metrics for each ticker using the `quantstats` library. Metrics include:
    *   **CAGR % (Compound Annual Growth Rate)**: Annualized return percentage.
    *   **Volatility % (ann.)**: Annualized standard deviation of returns, indicating risk.
    *   **Sharpe**: Risk-adjusted return (higher is better).
    *   **Sortino**: Risk-adjusted return focusing only on downside volatility.
    *   **Max Drawdown %**: The largest percentage drop from a peak to a trough.
    *   **Beta vs SPY**: Measures the stock's volatility relative to the benchmark.
    *   **Corr. vs SPY**: Correlation coefficient with the benchmark, indicating how closely the stock moves with it.
    The results are presented in a DataFrame, sorted by Sharpe ratio.
*   **`generate_tearsheets(returns, tickers, benchmark)`**: This function creates detailed HTML tearsheet reports for each individual stock. These reports are generated by `quantstats` and provide an extensive analysis of a stock's performance, risk, drawdowns, and more, relative to the benchmark. Each tearsheet is saved as an HTML file in the `TEARSHEET_DIR`.

#### Stage 5: Visualize Results

This stage creates visual summaries and exports combined reports for easy understanding.

*   **`build_dashboard(returns, metrics_df, tickers, benchmark)`**: This function generates a 4-panel executive dashboard plot using `matplotlib` and `seaborn`. The panels typically include:
    *   **Cumulative Returns**: Shows the growth of an initial investment over time for each stock and the benchmark.
    *   **Drawdown**: Illustrates the peak-to-trough declines.
    *   **Correlation Heatmap**: Visualizes the correlation between all analyzed stocks and the benchmark.
    *   **Risk vs Return Scatter**: Plots annualized volatility against CAGR, often indicating the efficient frontier.
    The dashboard is saved as a PNG image in the `OUTPUT_DIR`.
*   **`export_executive_summary(metrics_df, fundamentals_df)`**: This function combines the performance metrics and fundamental data into a single Excel workbook with two separate sheets (`Performance Metrics` and `Fundamentals`). This is ideal for sharing with stakeholders.

#### Main Pipeline Execution

*   **`main()`**: This function orchestrates the entire workflow by calling the functions defined in the preceding stages in the correct order. It first acquires and structures the data, then computes metrics and generates tearsheets, and finally creates the dashboard and executive summary.
*   **`if __name__ == "__main__": main()`**: This standard Python construct ensures that the `main()` function is called only when the script is executed directly (not when imported as a module), allowing the entire analysis pipeline to run.
